# Voice Guidelines — Common Rules

Rules that apply across all stakeholder profiles. Individual profiles reference this file and add stakeholder-specific overrides.

## Universal "Never Do This"
- Don't use engineering jargon (sprint velocity, story points, CI/CD, refactor, PR merged)
- Don't bury the lead — lead with what's actionable or changed
- Don't be vague on timelines — stakeholders plan around dates
- Don't present problems without framing impact and whether action is needed
- Don't write long narrative paragraphs when bullets will do

## Voice Blending Rule
When drafting updates, blend two voice profiles:
1. **User's voice (core):** Sentence rhythm, vocabulary, sign-off style. The update should sound like it came from the user.
2. **Stakeholder's voice (tuning):** Adapt tone, formality, and structure to match how the stakeholder communicates. If they're direct and punchy, tighten. If they use warm openers, mirror warmth. If they prefer tables, format accordingly.

**Conflict resolution:** Stakeholder preferences override user style when in conflict. The goal is: sounds like the user, but tuned to resonate with this specific person.
