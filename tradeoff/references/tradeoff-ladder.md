# Trade-off Ladder

Canonical reference lifted from CLAUDE.md "How I Make Decisions" section. Walk rungs in order. Stop early if a rung resolves the decision.

## Rungs

### 1. Find the 80/20
Can you get most of both without conceding the trade-off? Look for the option that collapses the dilemma entirely. If found, stop here.

### 2. First Principles
What are you actually optimizing for? Strip away surface framing and restate the objective. Options that don't serve the real objective are eliminated.

### 3. Reversibility
Favor the option you can undo. If one path is easy to walk back and the other is permanent, that's a strong tiebreaker.

### 4. Downside Avoidance
Which failure is worse? Avoid that one. Compare failure modes directly: "If option A fails, we get X. If option B fails, we get Y. Which is more survivable?"

## Evaluating Arguments (Trust Hierarchy)

When options are backed by competing arguments, rank evidence:
1. Evidence (data)
2. Logical coherence (does the reasoning hold?)
3. Mental model fit (does this match domain understanding?)
4. Track record (do I trust this person on this topic?)

Override mental models when evidence and logic are strong enough.

## Prioritization Default

Impact is the primary driver. Urgency, effort, dependencies, and gut all factor in but none override impact on their own. Weighed in context, not a checklist.

## Under Uncertainty

Default to the reversible choice and move. Fast, but sometimes skips the key unknown. The skill's Phase 2 ("Surface the Key Unknown") counters this bias.
